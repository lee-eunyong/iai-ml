# [정리] AI 모델 성능 평가: 혼동행렬과 평가지표

이 문서는 모델의 예측 결과와 실제 정답을 비교하여 모델의 성능을 다각도로 분석하는 방법을 정리한 문서입니다.

---

## 1. 혼동행렬 (Confusion Matrix)
혼동행렬은 분류 모델의 예측 결과가 실제값과 비교하여 얼마나 '혼동'되고 있는지 보여주는 표입니다.

### 행렬 구조 (2x2 예시)
보통 행(Row)은 **실제값**, 열(Column)은 **예측값**을 나타냅니다. (클래스는 0부터 오름차순 배치)

| | 예측: 0 (Negative) | 예측: 1 (Positive) |
| :--- | :---: | :---: |
| **실제: 0** | **TN** (True Negative) | **FP** (False Positive) |
| **실제: 1** | **FN** (False Negative) | **TP** (True Positive) |

### 주요 용어
- **TN (True Negative):** 실제 0을 0으로 올바르게 예측
- **FP (False Positive):** 실제 0인데 1로 잘못 예측 (제1종 오류)
- **FN (False Negative):** 실제 1인데 0으로 잘못 예측 (제2종 오류)
- **TP (True Positive):** 실제 1을 1로 올바르게 예측

---

## 2. 핵심 평가지표

### ① 정확도 (Accuracy)
전체 데이터 중에서 모델이 정답(0이든 1이든)을 맞춘 비율입니다.
- **공식:** `(TN + TP) / (TN + FP + FN + TP)`
- **특징:** 가장 직관적이지만, 데이터가 불균형할 경우(예: 0이 99%, 1이 1%) 모델의 실력을 제대로 평가하기 어렵습니다.

### ② 정밀도 (Precision)
모델이 **1이라고 예측한 것들** 중에서 실제로 1이었던 비율입니다.
- **공식:** `TP / (TP + FP)`
- **관점:** "모델의 예측이 얼마나 신중하고 정확한가?"
- **사례:** 스팸 메일 분류 (정상 메일을 스팸으로 오인하면 안 되는 경우)

### ③ 재현율 (Recall)
**실제로 1인 것들** 중에서 모델이 1이라고 맞춘 비율입니다.
- **공식:** `TP / (TP + FN)`
- **관점:** "모델이 실제 정답을 얼마나 놓치지 않고 찾아내는가?"
- **사례:** 암 진단, 화재 감지 (실제 환자나 화재를 놓치면 치명적인 경우)

### ④ F1-스코어 (F1-Score)
정밀도와 재현율의 **조화 평균**입니다.
- **공식:** `2 * (Precision * Recall) / (Precision + Recall)`
- **특징:** 정밀도와 재현율이 한쪽으로 치우치지 않고 균형을 이룰 때 높은 값을 가집니다. 데이터 불균형이 심할 때 모델 성능을 가장 잘 나타내는 지표입니다.

---

## 3. 요약 예시 분석
- **데이터:** 실제값 `[1, 0, 1, 1, 0, 1]` / 예측값 `[0, 0, 1, 1, 0, 1]`
- **혼동행렬 결과:**
[[2 0]
[1 3]]
- **해석:**
- **정확도:** 5/6 (83.3%) - 6개 중 5개 정답
- **정밀도:** 3/3 (100%) - 1이라고 한 건 다 맞음
- **재현율:** 3/4 (75%) - 실제 1인 4개 중 1개 놓침
- **F1-스코어:** 약 0.857 - 정밀도와 재현율의 균형 점수

---

## 4. 지표 선택 가이드
1. **정확도**를 기본으로 보되,
2. **틀린 예측(오보)**에 민감하다면 **정밀도**를,
3. **정답 누락(미탐)**에 민감하다면 **재현율**을,
4. 데이터가 편향되어 있거나 **종합적인 균형**이 중요하다면 **F1-스코어**를 확인합니다.
